# AGENTS.md

이 파일은 GitHub Copilot, Gemini CLI, Antigravity에서 공통으로 참고할 프로젝트 규칙이다. 무료 사용량이 남아 있는 도구를 번갈아 쓰더라도 같은 규칙으로 작업한다.

## 프로젝트

- 이름: 일정공지 에이전트
- 목표: 여러 학과 공지를 읽고, 날짜와 대상자를 정리한 뒤, 사용자별 해야 할 일을 안내한다.
- 기본 실행: `python schedule_agent.py`
- 기본 작업 파일: `schedule_agent.py`
- 입력 예시 파일: `sample_notices.txt`
- 주요 출력 파일: `output.md`, `output_user_guide.md`, `review_report.md`

## 에이전트 역할

1. 사실 추출 에이전트: 제목, 대상, 날짜, 해야 할 일, 준비물을 뽑는다.
2. 일정 분류 에이전트: 학년이나 사용자 유형별로 공지를 나눈다.
3. 안내문 작성 에이전트: 사용자별 해야 할 일을 Markdown으로 쓴다.
4. 검토자 에이전트: 날짜 누락, 대상 불명확, 공지에 없는 단정 표현을 확인한다.

## 작업 방식

1. 파일을 수정하기 전에 먼저 계획을 제시한다.
2. 한 번에 많은 파일을 바꾸지 않는다.
3. 기존 구조를 유지하면서 작은 단위로 고친다.
4. 오류가 나면 새 기능을 추가하지 말고 오류만 고친다.
5. 변경 후 실행 명령과 확인 결과를 보고한다.

## 금지

- 불필요한 패키지 설치
- API 키 하드코딩
- 수업 범위를 벗어난 복잡한 프레임워크 추가
- Playwright MCP 연결
- 사용자가 요청하지 않은 대규모 리팩터링

## Playwright CLI 사용

Playwright는 MCP가 아니라 CLI로만 사용한다.

```bash
playwright-cli open docs/week-10.html
```

브라우저 확인이 필요한 경우에만 사용하고, 확인 결과를 짧게 보고한다.
