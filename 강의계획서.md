## 강의 목표

본 과목을 이수하면 다음 역량을 갖추게 된다.

1. AI 협업 개발의 역할 분담(설계·검증·책임)을 이해하고, 검증 가능한 산출물을 만들 수 있다
2. MCP(Model Context Protocol)의 아키텍처를 이해하고, 도구·리소스를 설계·구현할 수 있다
3. 외부 API를 MCP 서버로 래핑하고, OAuth 2.1 및 A2A 프로토콜의 개념을 설명할 수 있다
4. LangChain과 LangGraph로 도구를 활용하는 에이전트와 상태 기반 워크플로우를 구현할 수 있다
5. 멀티에이전트 시스템(Supervisor·Handoff·평가)을 설계·구현하고, 단일 에이전트와의 차이를 판단할 수 있다
6. 멀티에이전트에 RAG·메모리·Human-in-the-loop를 통합하고, 배포·관측·비용 최적화를 수행할 수 있다

```
Phase 1. AI 협업 기초 (1~2주)
[1] AI 협업 패러다임 ─ 에이전틱 도구 전경, 바이브 코딩 한계, 검증 습관
       │
[2] 개발 환경 ──────── 재현 가능한 실행 환경 구축

Phase 2. MCP와 에이전트 프레임워크 (3~8주)
[3] MCP 개념 ────────── Tool·Resource·Prompt, STDIO 통신, 생태계
       │
[4] MCP 서버 구현 ──── 외부 API 래핑, 인증, 실패 처리, A2A 개요
       │
[5] LangChain 에이전트 ReAct 패턴, 구조화된 출력, Agents SDK
       │
[6] LangGraph ──────── State, 조건부 엣지, 반복 루프, LangSmith
       │
[7] 미니 프로젝트 ──── 날씨 요약 에이전트 (중간고사 대체)
       │
[8] 중간 점검 ──────── 1~7주 정리 + 멀티에이전트 사전 안내

Phase 3. 멀티에이전트 시스템과 통합 운영 (9~13주)
[9]  멀티에이전트 개념 단일 vs 멀티, 4대 패턴, 평가, 언제 쓰지 말 것
       │
[10] LangGraph 실습 ── Supervisor·Subgraph·Send·핸드오프·공유 메모리
       │
[11] 코딩 에이전트 ─── 트랙 선택(Copilot/Antigravity/Claude Code) + 공통 6단계
       │
[12] RAG·메모리 통합 ─ 검색 에이전트 + 작성 에이전트, 출처 검증
       │
[13] HITL·배포·관측 ── interrupt·Docker·LangSmith·비용 + 프로젝트 가이드

Phase 4. 최종 프로젝트 (14~15주)
[14] 집중 개발 ──────── 개인 프로젝트 + 교수 1:1 상담
       │

```

## 평가 방식

| 항목 | 비중 | 내용 |
|------|:----:|------|
| 출석 및 참여 | 10% | 15주 중 결석 3회 이상 시 F |
| 주차별 과제 | 30% | 2회차 실습 과제 (매주 제출, 12회) |
| 중간 평가 (미니 프로젝트) | 20% | 7주차 날씨 요약 에이전트로 대체 (기존 중간고사 폐지) |
| 최종 프로젝트 | 40% | 설계서 (5%) + 구현 (20%) + 발표·구두 평가 (15%) — 15주차 발표가 기말고사를 겸함 |

| 주차 | 과제 | 제출 형태 |
|:----:|------|----------|
| 1 | 체크리스트 파일 + AI 도구 비교 정리 | GitHub + 문서 |
| 2 | 재현 가능한 환경 구축 결과 | GitHub |
| 3 | MCP 아키텍처 다이어그램 + 최소 MCP 서버 | GitHub |

| 5 | LangChain 에이전트 + 도구 호출 결과 | GitHub |
| 6 | LangGraph 워크플로우 + LangSmith 트레이스 | GitHub |
| 8 | 미니 프로젝트 개선 + 멀티에이전트 사전 메모 | GitHub + 1쪽 메모 |
| 9 | 단일/멀티 비교 코드 2개 + 역할 분리표 | GitHub + 표 |

| 11 | 작업 정의서 + 트랙별 계획/결과 캡처 + 관찰 표 | GitHub or 저장소 |

| 13 | HITL + 비용 로그 + 프로젝트 설계서 초안 (Docker 가산점) | GitHub + 설계서 |

### 최종 프로젝트 (40%) — 기말고사 대체

기말고사를 폐지하고, 15주차 발표를 통해 최종 프로젝트로 통합 평가한다. 발표 후 5분 질의응답이 구두 평가를 겸한다.

**발표 구성** (10분 발표 + 5분 질의응답)
1. 문제 정의와 동기 (2분)
2. 시스템 아키텍처: MCP 서버 + 멀티에이전트(Supervisor) + RAG·HITL 구성 (3분)
3. 데모 시연 (3분)
4. 한계점 및 향후 과제 (2분)

**평가 기준**

| 항목 | 비중 | 설명 |
|------|:----:|------|
| MCP 서버 구현 완성도 | 20% | 도구/리소스 설계, 에러 처리, 테스트 |
| 멀티에이전트 워크플로우 | 30% | LangGraph Supervisor·핸드오프·공유 메모리 설계 |
| 통합 요소 (RAG·HITL·배포) | 20% | RAG 통합, HITL 승인, Docker 배포 중 2개 이상 |
| 발표 및 구두 평가 | 20% | 논리적 구성, 데모 완성도, 질의응답 |
| 코드 품질 | 10% | 구조, 문서화, Git 이력 |

**프로젝트 주제 예시**
1. 학과 공지사항 자동 요약 멀티에이전트 (크롤링 → 요약 → 검토)
2. 코드 리뷰 자동화 봇 (GitHub MCP + Supervisor + HITL 승인)
3. 논문 검색·분석 멀티에이전트 (검색 → 분석 → 보고서, 출처 추적 포함)
4. 고객 지원 멀티에이전트 (분류 → 답변 → 에스컬레이션 + 사람 승인)
5. 데이터 분석 멀티에이전트 (DB MCP + 분석 에이전트 + 검증 에이전트)
