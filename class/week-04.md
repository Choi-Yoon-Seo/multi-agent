## 4.1 왜 API를 MCP 서버로 감싸는가

- 외부 API는 서비스마다 요청 방식과 응답 형식이 다르다
- 에이전트가 외부 API를 직접 다루면 다음 문제가 발생한다
  - 인증 키 노출 위험
  - 에러 처리 방식이 제각각임
  - 같은 작업을 여러 프로젝트에서 반복 구현하게 됨
- MCP 서버로 감싸면 얻는 이점
  - 에이전트는 단순한 도구 인터페이스만 사용
  - 인증과 에러 처리를 서버 안으로 숨길 수 있음
  - 이후 다른 프로젝트에서도 재사용 가능

**한 줄 정리**

외부 API는 복잡하고 불안정할 수 있으므로, 에이전트가 직접 만지게 하기보다 MCP 서버가 중간에서 정리해 주는 편이 안전하다.

## 4.2 이번 주 실습 구조

이번 주 실습은 아래 흐름으로 진행한다.

1. 외부 API 하나를 고른다
2. 필요한 입력을 정한다
3. Python 함수로 API 호출 로직을 만든다
4. 예외 상황을 처리한다
5. MCP 도구로 노출한다
6. 에이전트에서 호출해 본다

권장 예제는 날씨 API다.

- 이유
  - 요청 구조가 단순함
  - 입력과 출력이 분명함
  - 실행 결과를 바로 확인할 수 있음

## 4.3 API 키 관리

### 잘못된 방식

- 코드에 API 키를 직접 넣는 방식
- 공개 저장소 업로드 시 즉시 노출될 수 있음

```python
api_key = "hard-coded-secret"
```

### 권장 방식

- `.env` 파일에 키 저장
- `.env.example`로 필요한 변수 이름만 공유
- 코드에서는 `os.getenv()`로 읽기

```python
from dotenv import load_dotenv
import os

load_dotenv()
api_key = os.getenv("OPENWEATHERMAP_API_KEY")

if not api_key:
    raise ValueError("OPENWEATHERMAP_API_KEY가 필요합니다.")
```

### 체크포인트

- `.env`는 `.gitignore`에 들어가 있는가
- `.env.example`에는 실제 키가 없는가
- 키가 없을 때 친절한 오류를 보여 주는가

## 4.4 실패 처리를 왜 넣어야 하는가

외부 API는 항상 정상 동작하지 않는다.

- 네트워크가 느릴 수 있음
- 잘못된 도시명을 넣을 수 있음
- 키가 잘못되었을 수 있음
- 서버가 일시적으로 응답하지 않을 수 있음

이번 주 실습에서는 아래 세 가지만 반드시 챙긴다.

### 1. 타임아웃

```python
response = requests.get(url, timeout=10)
```

- 무한 대기를 막는다

### 2. 재시도

- 모든 오류를 무한 재시도하지 않는다
- 일시적 네트워크 오류만 1~2회 다시 시도한다

### 3. 명확한 에러 메시지

- "실패"라고만 하지 말고 원인을 구분한다
  - 인증 실패
  - 입력 오류
  - 네트워크 오류
  - 타임아웃

## 4.5 최소 로깅

이번 주 로깅은 운영 수준이 아니라 디버깅 가능한 최소 수준이면 충분하다.

기록할 것:

- 어떤 입력이 들어왔는지
- 외부 API 호출이 성공했는지
- 실패했다면 어떤 종류의 실패인지

기록하지 말 것:

- API 키
- 민감한 전체 URL
- 필요 이상으로 긴 원본 응답 전체

```python
logger.info("weather request started: city=%s", city)
logger.warning("weather request failed: city=%s", city)
```

---

## 4.6 테스트 가능한 구조

실습에서 가장 많이 생기는 문제는 "API 호출 코드와 비즈니스 로직이 한 함수에 섞이는 것"이다.

권장 구조:

1. 외부 API를 호출하는 함수
2. 응답을 정리하는 함수
3. MCP 도구로 연결하는 함수

이렇게 나누면 좋은 점:

- API 없이도 응답 정리 함수를 테스트할 수 있음
- 에러 상황을 독립적으로 점검할 수 있음
- MCP 도구 바깥에서 먼저 디버깅할 수 있음

## 4.7 실습: 날씨 MCP 서버 만들기

### 실습 목표

- 도시 이름을 입력받아 현재 날씨를 반환하는 도구를 만든다

### 권장 기능

- 입력: 도시 이름
- 출력: 온도, 습도, 날씨 상태
- 예외 처리: 잘못된 도시명, 인증 실패, 타임아웃

### 최소 구현 흐름

1. `.env`에서 API 키 읽기
2. HTTP 요청 보내기
3. 응답 JSON에서 필요한 필드만 추출
4. 사람이 읽기 쉬운 결과로 정리
5. MCP 도구로 등록

### 실행 확인

- 정상 도시명으로 한 번 실행
- 없는 도시명으로 한 번 실행
- API 키 제거 상태로 한 번 실행

- 입력 오류와 네트워크 오류를 구분했다
- 최소 한 개 이상의 로그를 남겼다
- 정상 실행과 실패 실행을 모두 확인했다
