## 12.1 사람이 필요한 순간

다음 작업은 자동 실행보다 사람 확인이 더 적절하다.

- 데이터 삭제
- 외부 시스템 변경
- 민감한 정보 포함 응답
- 검증 실패가 반복된 경우

핵심 메시지:

- 자동화는 편리하지만, 책임까지 자동화되지는 않는다

---

## 12.2 승인 요청 메시지

좋은 승인 요청에는 아래가 포함되어야 한다.

- 무엇을 하려는가
- 어디에 영향을 주는가
- 왜 승인이 필요한가
- 어떤 선택지가 있는가

예시:

```text
작업: report.txt 파일을 덮어쓰려 합니다.
영향: 기존 파일 내용이 사라질 수 있습니다.
사유: 자동 생성 결과를 저장하려고 합니다.
선택: 승인 / 거부
```

---

## 12.3 최소 승인 워크플로우

권장 최소 흐름:

1. 작업 생성
2. 위험도 판단
3. 고위험이면 사람 승인 요청
4. 승인되면 실행
5. 거부되면 종료

여기서는 복잡한 권한 체계보다 "승인 지점이 있다"는 점이 중요하다.

---

## 12.4 실습: 간단한 승인 흐름

### 실습 목표

- 기존 워크플로우에 승인 단계를 한 번 넣는다

### 권장 사례

- 파일 저장 전 승인
- 메일 초안 전송 전 승인
- 외부 API 요청 전 승인

### 학생이 직접 확인할 것

- 승인 없이 바로 실행되던 작업이 어떻게 바뀌는가
- 메시지가 충분히 명확한가
- 거부 시 흐름이 안전하게 종료되는가
