# Week 14. 최종 프로젝트 집중 개발

> 학부 운영안 — 강의 없음, 개인 프로젝트 집중 + 교수 1:1 상담

---

## 학습 목표

- 9~13주차에서 배운 멀티에이전트·RAG·HITL·배포를 *동작하는 한 시스템*으로 통합한다
- 새 기능 추가보다 **동작 안정화**를 우선한다
- 15주차 발표(10분 + 5분 질의응답)에서 보여 줄 데모 흐름을 완성한다

---

## 이번 주 운영 원칙

- 강의 없음. 두 회차 모두 **개인 프로젝트 작업 시간**
- 교수는 1:1 상담 슬롯을 운영 (사전 예약, 회당 15분)
- 보여 줄 수 있는 **최소 데모를 먼저** 완성한 뒤 기능을 늘린다
- “안 되는 부분”을 숨기지 않는다 — 발표에서 *한계*를 말하는 게 더 좋은 평가를 받는다

## 14.1 한 주의 권장 흐름

| 일자(예시) | 권장 작업 |
|---|---|
| 1일차 | 13주차 설계서 초안 다듬기 → 핵심 기능 1개부터 동작 확인 |
| 2일차 | 멀티에이전트 골격(Supervisor + 2~3 에이전트) 완성 |
| 3일차 | 통합 요소 1개 추가 (RAG / HITL / Docker 중) |
| 4일차 | 통합 요소 2개째 추가 + 실패 케이스 처리 |
| 5일차 | 발표 슬라이드 초안 + 데모 시연 리허설 |
| 6일차 | 1:1 상담 피드백 반영 + 마무리 |
| 7일차 | 백업 시연(녹화 영상) 준비 |

## 14.2 개발 우선순위

다음 순서를 지킨다. 위 항목을 닫기 전에 아래로 가지 않는다.

1. **핵심 기능 1개 동작** — 한 사용자가 한 번 실행해서 결과가 나온다
2. **오류 처리** — 잘못된 입력에 명확한 메시지
3. **통합 요소 2개** (RAG·HITL·Docker 중) — 평가 기준 충족
4. **결과 정리** — 출처·로그·트레이스가 남는다
5. **발표 자료** — 슬라이드 + 데모 입력
6. **선택 기능** (있으면 좋은 것)

선택 기능이 핵심 기능보다 먼저 손대는 순간 14주차가 무너진다.

---

## 14.3 1:1 상담 활용법

상담 슬롯은 사전 예약제로 운영한다. 효율적으로 쓰려면:

### 가져올 것

- 현재 상태 한 줄 (“X는 되고 Y가 안 됨”)
- 막힌 지점의 **재현 코드** 또는 *명확한 에러 메시지*
- 시도해 본 두 가지 접근 + 왜 안 됐는지

### 받아 갈 것

- 막힌 지점의 우회 또는 해결 힌트
- 범위 조정 권고 (줄여야 할지 / 더 가도 될지)
- 발표 시연에서 강조할 포인트

“막연히 봐 주세요”는 15분이 모자란다. 위 양식으로 정리해서 오는 학생이 가장 많이 얻는다.

---

## 14.4 공통 점검 항목

15주차 발표 전 반드시 확인.

- [ ] 한 번 실행으로 데모가 끝까지 돌아간다
- [ ] `.env` 없이 시작해도 “키가 없다”는 *친절한 메시지*가 뜬다
- [ ] 핵심 출력에 출처/근거 또는 트레이스 ID가 남는다
- [ ] 멀티에이전트의 라우팅 로그를 보여 줄 수 있다
- [ ] 통합 요소 2개를 데모에서 실제로 보여 준다
- [ ] 데모 입력 3개를 미리 정해 두었다 (정상·경계·실패 사례 각 1)
- [ ] 백업으로 짧은 시연 영상을 녹화해 두었다 (라이브 실패 대비)

## 14.5 발표 슬라이드 권장 구성 (10장 내외)

| 장 | 내용 |
|---|---|
| 1 | 제목 + 한 문장 설명 |
| 2 | 풀려는 문제와 동기 |
| 3 | 단일 에이전트로 부족한 이유 (9주차 표 재활용) |
| 4 | 시스템 아키텍처 도식 |
| 5 | 에이전트 역할 분담 |
| 6 | 통합 요소 1 (RAG 등) — 어떻게 붙였는가 |
| 7 | 통합 요소 2 (HITL 또는 Docker) |
| 8 | 데모 (라이브 또는 영상) |
| 9 | 한계와 알려진 실패 |
| 10 | 다음 단계 (확장하면 무엇) |

---

## 14.6 흔한 막힘과 우회

| 증상 | 1차 우회 | 2차 우회 |
|---|---|---|
| Supervisor가 무한 루프 | `recursion_limit` 8~12 | Supervisor 규칙에 “재시도 1회만” |
| RAG가 엉뚱한 청크를 가져옴 | `chunk_size` 줄이기 | 검색 후 LLM 필터링 노드 추가 |
| Docker에서 vectorstore 못 찾음 | Dockerfile에 COPY 누락 확인 | 빌드 시 `docker build --no-cache` |
| LangSmith 트레이스 안 잡힘 | `.env` 재로드 | `LANGSMITH_TRACING=true`인지 확인 |
| HITL 입력 받기 어색함 | 터미널 input() 그대로 | 간단한 Streamlit UI 한 페이지 |
