## 11.1 환각이란 무엇인가

- 모델이 사실과 다르거나 근거 없는 내용을 그럴듯하게 말하는 현상

대표적인 문제:

- 사실 오류
- 논리 비약
- 출처 조작

학부 실습에서 중요한 점:

- 답이 자연스럽게 들린다고 바로 믿지 않는다
- 특히 문서 요약, 질의응답, 코드 설명에서 검증이 필요하다

---

## 11.2 간단한 검증 방법

### 1. 다시 묻기

- 같은 질문을 조금 다르게 다시 물어 본다

### 2. 출처 확인

- 답변의 근거 문서를 함께 본다

### 3. 규칙 확인

- 형식, 필수 항목, 금지 항목을 점검한다

이번 주에는 이 세 가지면 충분하다.

---

## 11.3 품질 게이트

품질 게이트는 "이 답변을 그냥 통과시켜도 되는가"를 검사하는 단계다.

예시:

1. 형식이 맞는가
2. 출처가 있는가
3. 자신 없는 표현이 많은가
4. 필수 항목이 빠지지 않았는가

게이트 실패 시 선택:

- 다시 생성
- 사람 확인
- 실패로 종료

---

## 11.4 실습: 작은 검증 워크플로우

### 실습 목표

- 생성 후 바로 출력하지 않고 검증 단계를 한 번 더 넣는다

### 권장 흐름

1. 초안 생성
2. 형식 검사
3. 출처 또는 근거 검사
4. 통과 시 출력
5. 실패 시 재시도 또는 사람 확인

### 학생이 직접 확인할 것

- 어떤 답변이 게이트를 통과하는가
- 어떤 답변이 걸러지는가
- 너무 엄격하면 정상 답변도 막히는가

---

## 제출물

- 검증 단계가 포함된 코드
- 통과 사례 1개
- 실패 사례 1개
- 짧은 메모
  - 어떤 기준으로 게이트를 만들었는가
