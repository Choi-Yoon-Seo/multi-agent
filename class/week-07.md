# Week 7. 미니 프로젝트 1

## 학습 목표

- 지금까지 배운 내용을 묶어 작은 자동화 프로젝트를 완성한다
- 문제 정의, 입력 설계, 오류 처리, 실행 검증을 스스로 정리한다
- 짧은 기술 설명과 데모를 통해 구현 내용을 설명한다

---

## 이번 주 운영 원칙

- 새로운 개념을 많이 추가하지 않는다
- 4~6주차에 배운 내용을 실제 산출물로 묶는 데 집중한다
- 프로젝트 규모는 작게, 완성도는 높게 가져간다

---

## 프로젝트 범위

다음 중 하나를 선택한다.

1. 외부 API를 래핑한 MCP 서버 만들기
2. 도구 호출 에이전트 만들기
3. LangGraph 기반 생성-검증-수정 워크플로우 만들기

권장 수준:

- 입력 1~2개
- 도구 1~3개
- 실패 처리 1개 이상
- 실행 결과 저장 또는 출력

권장하지 않는 수준:

- 여러 외부 시스템을 한 번에 연결하는 과도한 주제
- 멀티에이전트 구현
- 복잡한 운영 자동화

---

## 진행 단계

### 1. 문제 정의

- 무엇을 자동화할 것인가
- 입력은 무엇인가
- 출력은 무엇인가

### 2. 구조 설계

- 어떤 함수가 필요한가
- 외부 API 또는 도구는 무엇인가
- 어디에서 오류가 날 수 있는가

### 3. 구현

- 먼저 핵심 흐름을 작동시킨다
- 이후 로그와 에러 처리를 보완한다

### 4. 검증

- 정상 입력
- 실패 입력
- 경계 입력

### 5. 정리

- 실행 방법
- 결과 예시
- 한계

---

## 예시 주제

- 도시 이름을 받아 날씨 요약 파일 저장
- 간단한 학과 공지 요약 도구
- 초안 생성 후 검증과 수정을 반복하는 글쓰기 도우미
