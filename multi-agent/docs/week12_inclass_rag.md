# Week 12. 수업 실습: RAG 검색 에이전트 예시

## 1. 예시 과제

- 입력: 제공된 문서 3개
- 작업: 질문과 관련된 청크를 검색하고, 검색된 청크만 근거로 답변한다.
- 최종 출력: 답변 + 출처 + 출처 검증 결과

## 2. 역할 설계

| 역할 | 입력 | 작업 | 출력 | 검증 기준 |
|---|---|---|---|---|
| 검색 에이전트 | question | 벡터 저장소에서 관련 청크 검색 | retrieved_chunks | 질문과 관련 있는가 |
| 작성자 | question + retrieved_chunks | 청크 기반 답변 작성 | draft | 청크 밖 사실이 없는가 |
| 출처 검증 게이트 | draft + retrieved_chunks | 답변 문장과 청크 일치 확인 | PASS/FAIL | 모든 주장에 근거가 있는가 |

## 3. 공유 상태

- question: 사용자 질문
- retrieved_chunks: 검색된 문서 조각
- draft: 작성자 답변
- citations: 출처 목록
- grounding_result: PASS 또는 FAIL
- messages: 실행 로그

## 4. 예상 흐름

```text
[질문]
   ↓
[검색 에이전트]
   ↓
[작성자]
   ↓
[출처 검증 게이트]
   ↓
[최종 답변 + 출처]
```

## 5. PASS 예시

- 질문: 제공 문서에서 설명한 핵심 개념은 무엇인가?
- 기대 결과: 답변의 각 핵심 문장이 검색된 청크에 근거한다.
- 검증 결과: PASS

## 6. FAIL 예시

- 질문: 문서에 없는 최신 사건까지 포함해 설명해줘.
- 기대 결과: 작성자가 청크 밖 사실을 만들면 검증 게이트가 FAIL을 낸다.
- 후속 처리: 작성자는 “자료에 없다”는 문장으로 재작성한다.

## 7. 5문장 회고

1. RAG는 모델이 모르는 정보를 외부 문서에서 가져오게 한다.
2. 검색 청크 품질이 낮으면 좋은 답변을 만들기 어렵다.
3. 출처 검증 게이트는 답변이 검색된 문서에 근거하는지 확인한다.
4. 청크 크기가 너무 크면 관련 없는 정보가 섞이고, 너무 작으면 문맥이 끊긴다.
5. 13주차에는 PASS 이후에도 사람이 승인하는 단계를 추가할 수 있다.
