# Week 11. 수업 실습: 코딩 에이전트 위임 예시

## 1. 예시 작업

- 대상 폴더: `multi-agent/`
- 작업: 간단한 계산 함수에 입력 검증을 추가하고 테스트를 통과시킨다.
- 목적: 에이전트에게 바로 구현을 맡기지 않고, 계획 검토와 결과 검토를 분리한다.

## 2. 작업 정의서

```markdown
## Goal
calculator.core.add 함수가 숫자가 아닌 입력을 받으면 명확한 오류를 내도록 수정한다.

## Scope
- 수정 가능: multi-agent/code/calculator.py, multi-agent/code/test_calculator.py
- 수정 금지: .env, .venv, class/
- 변경 금지: 함수 이름 add

## Validation
- pytest multi-agent/code/test_calculator.py 통과
- 숫자 입력은 기존처럼 더한다
- 문자열 입력은 TypeError를 낸다
```

## 3. HITL 게이트

| 단계 | 사람이 확인할 것 | 통과 기준 |
|---|---|---|
| 계획 검토 | 수정 파일과 테스트 계획 | 범위를 벗어나지 않는다 |
| 결과 검토 | diff와 테스트 결과 | 요구사항과 테스트가 맞다 |

## 4. 관찰 표

| 항목 | 기록 |
|---|---|
| 사용 도구 | GitHub Copilot / Antigravity / Claude Code 중 선택 |
| 계획에서 좋았던 점 | 수정 파일과 테스트 파일을 먼저 제안했다 |
| 사람이 추가한 제약 | `.env`, `.venv`, `class/` 수정 금지 |
| 결과 검토에서 본 것 | diff, 테스트 결과, 오류 메시지 |
| 남은 위험 | 숫자처럼 보이는 문자열 처리 정책은 별도 결정 필요 |

## 5. 5문장 회고

1. 코딩 에이전트 위임은 작업 정의서가 명확할수록 안정적이다.
2. 계획 검토를 거치면 수정 범위를 미리 제한할 수 있다.
3. 결과 검토에서는 코드보다 테스트 결과와 diff를 함께 봐야 한다.
4. 자동 적용을 켜면 잘못된 파일 수정도 그대로 반영될 수 있다.
5. 10주차 Supervisor와 마찬가지로 사람은 중요한 분기에서 승인 역할을 한다.
